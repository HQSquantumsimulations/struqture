struqture_py.bosons.BosonHamiltonian

class struqture_py.bosons.BosonHamiltonian

These are representations of systems of bosons.

BosonHamiltonians are characterized by a BosonOperator to represent the hamiltonian of the spin system and an optional number of bosons.

Returns:

The new BosonHamiltonianSystem with the input number of bosons.

Return type:

self

Examples

import numpy.testing as npt
import scipy.sparse as sp
from qoqo_calculator_pyo3 import CalculatorComplex
from struqture_py.bosons import BosonHamiltonian, HermitianBosonProduct

system = BosonHamiltonian()
pp = HermitianBosonProduct([0], [0])
system.add_operator_product(pp, 5.0)
npt.assert_equal(system.current_number_modes(), 1)
npt.assert_equal(system.get(pp), CalculatorComplex(5))
npt.assert_equal(system.keys(), [pp])

dicke_boson_spin_mapping(number_spins_per_bosonic_mode)

Transforms the given bosonic object into a spin object using the mapping.

This mapping was developped by Juha Leppäkangas at HQS Quantum Simulations. The paper detailing the mapping, as well as its use in the context of open system dynamics, can be found at:

<https://arxiv.org/pdf/2210.12138>

The mapping is given by:

$ hat{b}_i^{dagger} hat{b}_i rightarrow sum_{j=1}^{N} hat{sigma}_+^{i,j} hat{sigma}_-^{i,j} $ $ hat{b}_i^{dagger} + hat{b}_i rightarrow frac{1}{root{N}} sum_{j=1}^{N} hat{sigma}_x^{i,j} $

For a direct mapping, N is set to 1. For a Dicke mapping, N > 1.

Parameters:

number_spins_per_bosonic_mode (int) -- The number of spins to represent each bosonic mode.

Returns:

The result of the mapping to a spin object.

Return type:

PauliOperator

Raises:

ValueError -- The boson -> spin transformation is only available for terms such as b†b or (b† + b).
